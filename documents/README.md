# HTML報告書テンプレート - PDF印刷検証プロジェクト (Node.js版)

ビジネス文書として適切な形式の報告書HTMLテンプレートと、Node.jsローカルサーバーでの印刷検証環境です。

## 📋 プロジェクト概要

このプロジェクトは、議事録、アジェンダ、報告資料などのビジネス文書をHTMLで作成し、ブラウザからPDFとして出力するためのテンプレート集です。

### 主な機能

- ✅ 7種類の豊富なテンプレート（標準、週次、ダッシュボード、正式、モダン、エグゼクティブ、アジャイル）
- ✅ ビジネス文書として適切なレイアウト
- ✅ PDF印刷に最適化されたスタイル
- ✅ ブラウザで直接編集できるコンテンツ領域
- ✅ 画像挿入対応
- ✅ テーブル、リスト、プログレスバー、ステータスバッジ
- ✅ A4サイズ最適化
- ✅ Meiryo UIフォント優先
- ✅ Node.jsローカルサーバーで簡単にホスト可能

## 📝 テンプレート一覧

### 1. 標準報告書 (standard.html)

最も汎用的な報告書テンプレート。8つの標準セクション（表紙、概要、目標、進捗、課題、成果物、今後の予定、添付資料）を持つ。月次報告や詳細なプロジェクト報告に最適。

### 2. 週次報告書 (compact.html)

週次報告に最適化されたコンパクトな1ページ版。サマリーカード、日次活動トラッキング、シンプルな課題管理を装備。

### 3. ダッシュボード (dashboard.html)

プロジェクト全体を一目で把握できるメトリクス重視のレイアウト。KPIカード、2カラムレイアウト、プログレスバーによる視覚的な進捗表示。

### 4. 正式報告書 (formal.html)

フォーマルな文書形式。章番号付き構成、承認・確認欄、表紙・目次対応。公式な報告書や提案書に最適。

### 5. モダン報告書 (modern.html)

グラデーションとグラスモーフィズムを使った最新デザイン。ヒーローカバー、プログレスリング、タイムライン形式で視覚的に魅力的。

### 6. エグゼクティブ報告書 (executive.html)

経営層向けのプレミアムデザイン。ダークモード表紙、KPIダッシュボード、インサイトカード、エグゼクティブサマリーを装備。

### 7. スプリント報告書 (agile.html)

アジャイル・スクラム開発向け。前スプリントアクションアイテムフォローアップ、バーンダウンチャート、ベロシティトレンド、ストーリーカード進捗、レトロスペクティブ（KPT）を完備。

## �️ 初回セットアップ（キッティング）

新しい環境でこのプロジェクトを使い始める手順です。

### 1. Node.jsのインストール確認

このプロジェクトにはNode.js 12以上が必要です。

```powershell
# Node.jsがインストールされているか確認
node --version
```

- ✅ バージョンが表示される（例: `v18.12.0`）→ 次のステップへ
- ❌ エラーが表示される → Node.jsをインストール: https://nodejs.org/

### 2. プロジェクトファイルの配置

このプロジェクトを任意の場所に配置します。

```powershell
# 例: デスクトップに配置する場合
cd "c:\Users\YOUR_USERNAME\Desktop\src\html検証"
```

### 3. 依存関係のインストール

このプロジェクトは**Node.js標準ライブラリのみ**を使用しているため、**npm installは不要**です。

### 4. 動作確認

サーバーを起動して、正常に動作することを確認します。

```powershell
# サーバーを起動
node server.js
```

または、`start-server.bat` をダブルクリック。

### 5. ブラウザで確認

http://localhost:8000/index.html にアクセスして、テンプレート一覧が表示されることを確認します。

### ✅ セットアップ完了

これで環境構築は完了です！サーバーを停止（Ctrl + C）して、次回からは通常の起動方法で使用できます。

## 📄 PDFへの印刷方法

### Chrome / Microsoft Edge の場合

1. テンプレート一覧（http://localhost:8000/index.html）から使用したいテンプレートを開く
2. 右上のメニュー（⋮）→ **印刷** を選択、または `Ctrl + P`
3. 印刷設定で以下を選択：
   - **送信先**: PDFに保存
   - **ページ**: すべて
   - **レイアウト**: 縦
   - **余白**: デフォルトまたはなし
   - **背景のグラフィック**: ✅ オン（重要！）
4. **保存** をクリックしてPDFとして保存

### Firefox の場合

1. `Ctrl + P` で印刷ダイアログを開く
2. **プリンター**: Microsoft Print to PDF
3. **背景の色と画像を印刷**: ✅ オン
4. **印刷** をクリック

## ✏️ 報告書の編集方法

### コンテンツの編集

1. ブラウザでテンプレートを開く（例: http://localhost:8000/templates/standard.html）
2. 編集したい箇所をクリック
   - 編集可能な要素にはホバー時に点線の枠が表示されます
3. 直接テキストを編集
4. 編集が完了したら、PDFとして保存（Ctrl + P）
5. 保存したPDFを `outputs/` フォルダの該当サブフォルダに配置

### HTMLファイルを直接編集

より高度な編集が必要な場合：

```powershell
# VS Codeで開く
code templates/standard.html
```

主な編集ポイント：

- `contenteditable="true"` 属性がある要素は、ブラウザで直接編集可能
- セクションの追加・削除は HTMLタグを編集
- スタイルの変更は `<style>` タグ内または `assets/css/print.css` を編集

### フォルダの役割

- **templates/**: 再利用可能なHTMLテンプレートを保管。編集前のクリーンな状態を維持。
- **outputs/**: PDFとして印刷・保存した実際の報告書を保管。種類別にサブフォルダで整理。
- **assets/**: 画像、CSS、ロゴなど複数のテンプレートで共有するリソースを保管。

## 🎨 カスタマイズガイド

### ロゴの変更

1. `assets/logos/` フォルダに会社ロゴを配置（例: `company-logo.png`）
2. テンプレートファイルの表紙部分を編集：

```html
<!-- 変更前 -->
<div class="logo-placeholder">LOGO</div>

<!-- 変更後 -->
<img
  src="../assets/logos/company-logo.png"
  alt="会社ロゴ"
  style="width: 150px;"
/>
```

### 配色の変更

テンプレートファイル内の `<style>` タグで主要な色を変更：

```css
/* メインカラー（グラデーション開始） */
#667eea → 任意の色コード

/* メインカラー（グラデーション終了） */
#764ba2 → 任意の色コード
```

### 画像の追加

```html
<div class="image-container">
  <img src="../assets/images/your-image.png" alt="説明" />
  <p class="image-caption">図X: 画像の説明</p>
</div>
```

### フォントの変更

すべてのテンプレートは以下のフォントスタックを使用：

```css
font-family: "Meiryo UI", "Meiryo", "MS PGothic", "MS Gothic", sans-serif;
```

優先順位を変更する場合は、各テンプレートの `<style>` タグ内で編集してください。

## 🔧 トラブルシューティング

### サーバーが起動しない

**エラー: Node.jsがインストールされていません**

→ Node.js をインストール: https://nodejs.org/

**エラー: ポート 8000 は既に使用されています**

→ 別のプログラムがポート8000を使用しています。以下のいずれかを実行：

1. そのプログラムを停止する
2. 別のポートを使用（`server.js` の `PORT = 8000` を変更）

### PDFで背景色が印刷されない

→ 印刷設定で「背景のグラフィック」を有効にしてください

### 画像が表示されない

1. 画像ファイルが `assets/images/` フォルダに存在するか確認
2. ファイル名が正しいか確認（大文字小文字を区別）
3. テンプレートからの相対パス（`../assets/images/`）が正しいか確認
4. ブラウザの開発者ツールでエラーメッセージを確認（F12キー）

### テンプレートのリンクが開けない

→ サーバーが起動していることを確認してください。
→ URLが正しいことを確認してください（`http://localhost:8000/templates/`）。

2. 進捗状況を「想定スケジュール」に変更
3. 成果物を「期待される成果」に変更
4. PDFとして保存
