1.リポジトリの推奨構成

GitHub 上で Skill を効果的に共有・管理するためには、統一されたディレクトリ構成が重要です。以下は推奨される構成です。

1.1 公開リポジトリのディレクトリレイアウト

agent-skills/ ├── skills/ │ ├── react-best-practices/ │ │ ├── SKILL.md # 必須 │ │ ├── metadata.json # 推奨 │ │ ├── AGENTS.md # 推奨 │ │ ├── README.md # 推奨 │ │ ├── rules/ # オプション │ │ │ ├── async-*.md │ │ │ ├── bundle-*.md │ │ │ └── ... │ │ └── scripts/ # オプション │ │ ├── deploy.sh │ │ └── validate.sh │ │ │ ├── deploy-to-vercel/ │ │ ├── SKILL.md │ │ ├── scripts/ │ │ │ └── deploy.sh │ │ └── resources/ │ │ ├── deploy-codex.sh │ │ └── deploy.sh │ │ │ └── composition-patterns/ │ ├── SKILL.md │ └── rules/ │ ├── packages/ # パッケージ化用 │ └── react-best-practices-build/ │ ├── package.json │ └── build.ts │ ├── documents/ # ドキュメント │ ├── README.md │ └── assets/ │ ├── diagrams/ │ └── images/ │ ├── README.md # リポジトリ概要 ├── AGENTS.md # Skill 作成ガイド ├── package.json # 全体の package.json └── .gitignore

1.2 各フォルダの役割

表1：ディレクトリ別の役割
フォルダ名 必須 説明
skills/ ✅ すべての Skill が格納されるルートディレクトリ
skills/[skill-name]/ ✅ 個別 Skill のディレクトリ（kebab-case で命名）
rules/ 推奨 複雑な Skill の場合、ルール ファイルを分割
scripts/ オプション Skill から呼び出される実行スクリプト
packages/ オプション ビルドツール・パッケージング用
documents/ 推奨 このドキュメント群など、ドキュメント専用

2.Skill のパッケージング（Zip 化）

Skill を配布するには、フォルダ全体を Zip ファイルに圧縮します。以下の手順に従ってください。

2.1 Zip ファイル作成手順

  1. ターミナルで skills ディレクトリへ移動
    cd agent-skills/skills
  2. 個別 Skill を Zip 化
    # macOS / Linux zip -r react-best-practices.zip react-best-practices/ # Windows PowerShell Compress-Archive -Path react-best-practices -DestinationPath react-best-practices.zip # 確認 ls -lh react-best-practices.zip
  3. 複数 Skill をまとめて Zip 化
    # すべての Skill を 1 つの Zip に zip -r agent-skills.zip react-best-practices/ deploy-to-vercel/ composition-patterns/ # ファイルサイズ確認（大きすぎないか） du -sh agent-skills.zip

2.2 Zip ファイルの確認

# Zip 内容を確認（解凍せずに） unzip -l react-best-practices.zip # 出力例: # Archive: react-best-practices.zip # Length Date Time Name # --------- ---------- ----- ---- # 1234 2026-04-03 10:00 react-best-practices/SKILL.md # 5678 2026-04-03 10:00 react-best-practices/metadata.json # ...

2.3 ベストプラクティス

項目 推奨方法
ファイルサイズ Skill 1 つで 1 MB 以下を目安（スクリプトと rules は別途確認）
最上位フォルダ Zip 解凍後、skill-name/ フォルダが最上位になること
ファイル名 Zip 名 = skill ディレクトリ名（例：react-best-practices.zip）
除外ファイル node_modules, .git, __pycache__ などは除外

3.GitHub へのアップロード・公開

Skill を GitHub に公開し、チーム内やコミュニティで共有するための手順です。

3.1 新規リポジトリ作成

  1. GitHub で新規リポジトリ作成
    GitHub にログイン → 「New repository」 → リポジトリ名を入力
    例）agent-skills
  2. ローカルで初期化
    cd agent-skills # リポジトリ初期化 git init # 最初のコミット git add . git commit -m "Initial commit: Add agent skills" # リモート設定 git remote add origin https://github.com/[username]/agent-skills.git # ブランチ名を main に変更（GitHub がデフォルト） git branch -M main # 公開 git push -u origin main

3.2 Releases タブでの Zip 配布

  1. GitHub リポジトリの「Releases」タブを開く
  2. 「Create a new release」をクリック
  3. タグ名と説明を入力
    Tag version: v1.0.0 Release title: React Best Practices v1.0.0 Description: ## 変更内容 - 初版リリース - 65個のパフォーマンスルールを収録 - CLI の自動検証機能を実装
  4. Zip ファイルをアップロード
    「Attach binaries」エリアに react-best-practices.zip をドラッグアンドドロップ
  5. 「Publish release」をクリック

4.ユーザー向けインストール手順

配布したい Skill のインストール方法を、初心者向けに説明します。

4.1 方法1：CLI を使用（最も簡単）

# グローバルにインストール npm install -g agent-skills # パッケージをインストール agent-skills install react-best-practices # 確認 agent-skills list # 出力: react-best-practices (v1.0.0)

4.2 方法2：手動ダウンロード

  1. GitHub Releases から react-best-practices.zip をダウンロード
  2. Zip を解凍
    unzip react-best-practices.zip
  3. .claude/skills/ にコピー
    # macOS / Linux cp -r react-best-practices ~/.claude/skills/ # Windows PowerShell Copy-Item -Recurse -Path react-best-practices -Destination $env:USERPROFILE\.claude\skills\
  4. VS Code を再起動して確認

4.3 方法3：GitHub から直接クローン

# リポジトリをクローン git clone https://github.com/[username]/agent-skills.git # Skill フォルダをコピー cp -r agent-skills/skills/react-best-practices ~/.claude/skills/
