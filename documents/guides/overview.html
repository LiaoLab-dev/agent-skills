スキル1: React Best Practices

1️⃣

React Best Practices

優先度: 高

Vercel エンジニアリングによる React と Next.js のパフォーマンス最適化ガイドラインです。

  • ルール数: 65個（8カテゴリー）
  • Waterfalls 排除（CRITICAL）
  • バンドルサイズ最適化（CRITICAL）
  • サーバー/クライアント最適化
  • 再レンダリング最適化

📊 8つのカテゴリー

優先度 カテゴリー 説明 インパクト
1 Waterfalls 排除 非同期処理の逐次実行を排除 CRITICAL
2 Bundle 最適化 バンドルサイズの最小化 CRITICAL
3 Server パフォーマンス サーバー側の処理最適化 HIGH
4 Client データ取得 クライアント側 API 呼び出し最適化 MEDIUM-HIGH
5 Re-render 最適化 不要な再レンダリング排除 MEDIUM
6 Rendering パフォーマンス DOM レンダリング最適化 MEDIUM
7 JavaScript 最適化 JS コード実行効率化 LOW-MEDIUM
8 Advanced パターン 高度な最適化テクニック LOW

✅ 使用場面

  • 新しい React コンポーネント/Next.js ページを作成する
  • データ取得の実装（クライアント/サーバー側）
  • パフォーマンス問題についてコードをレビューする
  • バンドルサイズや読み込み時間を最適化したい

スキル2: Composition Patterns

2️⃣

Composition Patterns

優先度: 中

スケーラビリティを備えた React コンポジションパターン。複合コンポーネント、ステート昇格を通じてブール型プロップの増殖を回避します。

  • ルール数: 20+個（4カテゴリー）
  • ブール型プロップ削減
  • 複合コンポーネントパターン
  • 状態管理ベストプラクティス
  • React 19 新 API 対応

🚨 解決する問題

❌ ブール型プロップ増殖の悪例:
<Button variant="primary" size="lg" disabled={disabled} loading={isLoading} icon="..." />
→ テストケースが 2^n に増加、保守困難
✅ 複合パターンの良例:
<Button>
  <Button.Icon />
  <Button.Label>Click</Button.Label>
</Button>
→ シンプル、柔軟、テスト容易

📋 4つのカテゴリー

優先度 カテゴリー 説明
HIGH Component Architecture コンポーネント設計の基本原則、ブール型プロップ回避
MEDIUM State Management 複合コンポーネント内の状態管理効率化
MEDIUM Implementation Patterns 実装時の具体的パターン（明示的バリアント等）
MEDIUM React 19 APIs React 19 以降の新機能対応

✅ 使用場面

  • 多くのブール型プロップを持つコンポーネントをリファクタリングする
  • 再利用可能なコンポーネントライブラリを構築する
  • 柔軟なコンポーネント API を設計する
🎨

スキル3: Web Design Guidelines

3️⃣

Web Design Guidelines

優先度: 高

UI コードが Web インターフェースのベストプラクティスに準拠しているか監査。アクセシビリティ、パフォーマンス、UX をカバーする 100+ ルール。

  • ルール数: 100+個（11監査領域）
  • アクセシビリティ（A11y）対応
  • キーボードナビゲーション
  • パフォーマンス最適化
  • ダークモード対応

🔍 11の監査領域

# 監査領域 説明
1 アクセシビリティ ARIA ラベル、セマンティック HTML、スクリーンリーダー対応
2 フォーカス状態 見えるフォーカスインジケーター、focus-visible
3 フォーム 入力検証、エラーハンドリング、ラベル関連付け
4 アニメーション prefers-reduced-motion 対応、GPU 加速
5 タイポグラフィ スマートクォート、省略記号、正確な表記
6 画像 alt 属性、遅延ロード、最適なフォーマット
7 パフォーマンス 仮想化、content-visibility、リソースヒント
8 ナビゲーション URL が状態を反映、ディープリンク対応
9 ダークモード color-scheme、prefers-color-scheme 対応
10 タッチ/インタラクション touch-action、48x48px タップターゲット
11 ロケール/i18n Intl API、多言語対応

✅ 使用場面

  • UI をレビューして改善提案してほしい
  • アクセシビリティをチェックしてほしい
  • デザインの品質監査が必要
  • UX をベストプラクティスに対して確認したい

❓ よくある質問

Q: 6 つすべてのスキルを導入する必要があるか？

いいえ。優先度に基づいて選択してください。最小限ラインは React Best Practices + Deploy to Vercel から始めることをお勧めします。

Q: スキル間に依存関係または競合があるか？

いいえ。各スキルは独立しており、複数のスキルを同時に活用できます。むしろ、複数スキルの組み合わせで最大の効果が発揮されます。

Q: スキルなしで AI エージェントは動くのか？

はい。スキルはオプション機能です。ただし、スキルを追加することで、エージェントがより精度の高い提案をしてくれます。

Q: 自分でカスタムスキルを作成できるか？

はい。Agent Skills フォーマットに従うことで、カスタムスキルを作成・配布できます。
