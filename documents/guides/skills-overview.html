1.Agent Skillsとは

Agent Skills は、AI コーディング エージェント（GitHub Copilot、Claude Code、Cursor など）の能力を拡張するための「道具箱」です。標準化されたテキストフォーマット（SKILL.md）を使うことで、エージェントに特定の知識やタスク実行能力を追加できます。

1.1 実生活の例で理解する

従来の使い方：

  • 「GitHub Copilot に、React コードを書いてほしい」と頼む
  • → Copilot が「一般的な」React の知識で対応

Skill を使った使い方：

  • 「Skill を読み込んだ Copilot に、React コードを書いてほしい」と頼む
  • → Copilot が「Vercel の 10 年のノウハウを詰め込んだ React Best Practices」を忠実に守ってコード生成

1.2 何ができるのか

  • パフォーマンス最適化： React/Next.js のベストプラクティスをエージェントに学習させる
  • デプロイ自動化： Vercel へのデプロイを 1 コマンドで実行
  • ガイドライン遵守： Web アクセシビリティやデザイン基準を AI に守らせる
  • 業務自動化： カスタム Skill で、あなたのチーム特有の作業ルールを AI に教える

1.3 なぜ必要なのか

AI エージェントは優れていますが、企業やチーム特有のノウハウや規制に自動で従うわけではありません。Skill を使うことで：
  • ✅ AI の出力品質が飛躍的に向上
  • ✅ チームの作業ルールが自動で守られる
  • ✅ コード レビューの手間が減る
  • ✅ 一貫性のあるプロジェクト構造を保つ

2.主な概念

2.1 オンデマンド読み込み（効率的なコンテキスト管理）

Skill は 必要なときだけ読み込まれる ため、AI エージェントのコンテキスト（メモリ）を無駄に消費しません。

ユーザーのリクエスト ↓ エージェント：「このタスクはどのSkillに関係するか？」 ↓ 一致するSkillの SKILL.md だけを読み込み ↓ エージェントが作業を実行

2.2 エージェントの自動起動メカニズム

Skill に含まれる description フィールドが「起動トリガー」として機能します。

【SKILL.md の定義】 name: deploy-to-vercel description: "Deploy my app" ↑ （トリガードキュメント） 【ユーザーの指示】 User: "Deploy my app" 【Copilot の動作フロー】 1. 指示を受け取る ↓ 2. "Deploy my app" テキストと照合 ↓ 3. deploy-to-vercel Skill の description にマッチ！ ↓ 4. Skill を自動ロード ↓ 5. デプロイ処理開始

2.3 複数エージェントでの互換性

Skill は 標準化フォーマット に従っているため、すべてのエージェントで動作します。

エージェント 対応状態 説明
GitHub Copilot ✅ 対応 VS Code 上で動作
Claude Code ✅ 対応 ローカル開発環境
Cursor ✅ 対応 AI ネイティブエディタ
claude.ai ✅ 対応 ブラウザ上で動作
カスタムエージェント ✅ 対応 独自実装可能

2.4 VS Code での読み込み位置カスタマイズ

VS Code（GitHub Copilot）では、デフォルトで ~/.claude/skills/ から Skill を検索しますが、.vscode/settings.json の chat.agentSkillsLocations 設定で任意の位置を指定できます。

例： リポジトリの .github/skills から読み込む場合

// .vscode/settings.json { "chat.agentSkillsLocations": [ "${workspaceFolder}/.github/skills", "${userHome}/.claude/skills" ] }

メリット：

  • ✅ リポジトリ内の Skill を Git で管理可能
  • ✅ チーム全員が同じ Skill を使用
  • ✅ 任意のディレクトリ構成に対応

3.Skill の構造

3.1 典型的なディレクトリレイアウト

agent-skills/ ├── skills/ # Skill が格納されるフォルダ │ ├── react-best-practices/ # Skill 1 │ │ ├── SKILL.md # 必須：スキル定義ファイル │ │ ├── metadata.json # メタデータ │ │ ├── AGENTS.md # 詳細ドキュメント │ │ └── rules/ # 個別ルール │ │ │ ├── deploy-to-vercel/ # Skill 2 │ │ ├── SKILL.md │ │ └── scripts/ # 実行スクリプト │ │ │ └── composition-patterns/ # Skill 3 │ ├── SKILL.md │ └── rules/ │ └── README.md

3.2 各ファイルの役割

表1：Skill ファイル構成
ファイル名 必須 説明
SKILL.md ✅ 必須 Skill の定義ファイル。エージェントが最初に読むもの。概要、使用方法、引数などを記載。
metadata.json 推奨 バージョン、著者、作成日、参考リソースなどのメタデータ。
AGENTS.md 推奨 SKILL.md の詳細版。すべてのルール、例、説明を集約。AI エージェントが参考にする。
scripts/ オプション SKILL.md から呼び出される Bash スクリプト。実際の処理を担当。
rules/ オプション 複雑な Skill の場合、ルールを個別ファイルに分割。段階的に読み込み可能。

3.3 SKILL.md の最小構成

--- name: my-skill description: "1行の説明（トリガー用）" metadata: author: "Your Name" version: "1.0.0" --- # My Skill [詳細な説明] ## When to Apply このスキルを使う場面 ## How It Works [処理の流れ] ## Usage ```bash bash /path/to/script.sh [args] ```

4.主要概念の実例

4.1 Skill の自動起動の例

ユーザーの指示：

User: "このReactコンポーネントを最適化してほしい" ↓ Copilot: 1️⃣ 指示を分析 → 「React最適化」タスク 2️⃣ react-best-practices Skill の description と照合 3️⃣ マッチ！ SKILL.md を読み込み 4️⃣ ベストプラクティスルールに従ってコード最適化 5️⃣ 改善提案を提示

4.2 複数 Skill の協力例

ユーザーの指示：

User: "UI コンポーネントを作って、デプロイして" ↓ Copilot: 1️⃣ composition-patterns Skill ロード → UI 設計 2️⃣ react-best-practices Skill ロード → パフォーマンス最適化 3️⃣ deploy-to-vercel Skill ロード → デプロイ実行 4️⃣ すべてのベストプラクティスを統合したコンポーネント生成 5️⃣ 自動デプロイ

4.3 コンテキスト効率化の効果

表2：コンテキスト使用量の比較
項目 Skill なし Skill を使用
起動時に読み込む情報 すべての Skill 詳細 Skill の説明のみ
コンテキスト消費 大（100KB+） 最小（5KB）
起動速度 遅い 高速
必要なときだけ読み込み ❌ 不可 ✅ 可能
