Skill ディレクトリの設定

Skill をどこに配置するか

VS Code が Skill ファイルを認識するための推奨ディレクトリ構成を説明します。

推奨ディレクトリ構成

my-project/ ├── .github/ │ ├── skills/ │ │ ├── react-best-practices/ │ │ │ ├── SKILL.md │ │ │ ├── metadata.json │ │ │ └── AGENTS.md │ │ ├── deploy-to-vercel/ │ │ │ ├── SKILL.md │ │ │ └── scripts/ │ │ └── composition-patterns/ │ │ ├── SKILL.md │ │ └── rules/ │ ├── workflows/ │ └── ISSUE_TEMPLATE/ ├── .vscode/ │ └── settings.json ├── src/ └── package.json

ディレクトリ配置のステップ

  • Step 1
    .github ディレクトリを作成
    # macOS / Linux mkdir -p .github/skills # Windows PowerShell New-Item -Path .github\skills -ItemType Directory
  • Step 2
    agent-skills をコピー
    # agent-skills リポジトリから skills をコピー cp -r agent-skills/skills/* .github/skills/ # 確認 ls .github/skills/ # 出力例: # react-best-practices/ deploy-to-vercel/ composition-patterns/
  • Step 3
    VS Code で認識確認
    VS Code を開く → コマンドパレット（Ctrl+Shift+P）→ 「Copilot: Open Slash Commands」を実行 → 登録された Skill が表示されるかチェック

copilot-instructions.md の設定

copilot-instructions.md とは

GitHub Copilot に与える「プロジェクト専用の指示書」です。プロジェクトコンテキスト（アーキテクチャ、コーディング規則、技術スタックなど）を定義することで、Copilot を「優秀なバイト」から「頼れる専属アシスタント」へと育て上げます。.github ディレクトリに配置するだけで自動認識されます。

copilot-instructions.md の構成（7つの要素）

# GitHub Copilot Instructions for Project X ## 1. 前提条件 - **回答は必ず日本語でしてください。** - コードの変更をする際、変更量が 200 行を超える可能性が高い場合は、事前に「この指示では変更量が 200 行を超える可能性がありますが、実行しますか?」と確認をとってください。 - 何か大きい変更を加える場合、まず何をするのか計画を立てた上で提案してください。 ## 2. プロジェクト概要 **Agent Skills Documentation Portal** は、初心者向けに Agent Skills と GitHub Copilot の統合、設定方法、実装例を解説するドキュメンテーションサイトです。 - **目的**: 開発者が GitHub Copilot + Agent Skills を効率的に導入・活用できるようにサポート - **対象ユーザー**: Agent Skills 初心者、GitHub Copilot ユーザー ## 3. 技術スタック | カテゴリ | 使用技術 | |--------|---------| | **フロントエンド** | HTML5, CSS3, Flexbox, Glassmorphism | | **出力形式** | A4 最適化 HTML (印刷・PDF対応) | | **主要ライブラリ** | VS Code API, GitHub Copilot API | | **設定ファイル形式** | YAML frontmatter, Markdown | ## 4. ディレクトリ構成 ``` .github/ ├── skills/ # Agent Skills ディレクトリ │ ├── react-best-practices/ │ ├── deploy-to-vercel/ │ └── composition-patterns/ ├── copilot-instructions.md # このファイル └── workflows/ ``` ## 5. アーキテクチャ・設計指針 - **SKILL.md フォーマット**: YAML frontmatter + Markdown の標準形式を厳密に守る - **ドキュメント構造**: A4 ページで管理し、セクションは明確に分割 - **一次情報の強調**: すべてのドキュメントに引用元・参考資料を明記 - **Visual Hierarchy**: CSS グラディエント、セクションアイコン、テーブル レイアウトで情報を階層化 ## 6. テスト・検証ポリシー - **ブラウザ確認**: http://localhost:8000 で各 HTML ドキュメントをレンダリング確認 - **コンテンツ検証**: - すべての内部リンク(.github/skills など)が正しいか確認 - Qiita 記事、GitHub 公式ドキュメントなど引用元の正確性チェック - **スタイル確認**: 印刷プレビューでも正常にレンダリングされるか確認 ## 7. アンチパターン（避けるべき記述） - ❌ 不正確なコマンド例 → ✅ macOS/Linux/Windows すべての正確なコマンド記載 - ❌ 不正確なコマンド例 → ✅ Linux/Windows すべての正確なコマンド記載 - ❌ 一次情報なしの説明 → ✅ 必ず引用元を明記 - ❌ .instructions.md という古い名称 → ✅ copilot-instructions.md を使用

設定ファイルの作成手順

  1. ファイルを作成
    # .github/copilot-instructions.md を新規作成 # ファイル配置: .github/copilot-instructions.md # 特別な設定不要、配置するだけで自動認識されます
  2. VS Code の認識確認
    Copilot チャット → 「@instructions」でファイルが参照されているかチェック
    もしくは Copilot がプロジェクトコンテキストを参照し始めるので、提案内容に反映されているか確認
  3. 反復改善
    使用しながら、Copilot の提案がプロジェクト要件と合致しているか様子を見ます。 必要に応じてファイルを更新・追加します。

ファイル配置例

my-project/ ├── .github/ │ ├── skills/ # Agent Skills │ │ ├── react-best-practices/ │ │ ├── deploy-to-vercel/ │ │ └── composition-patterns/ │ ├── copilot-instructions.md ← プロジェクト指示書 │ └── workflows/ ├── .vscode/ │ └── settings.json ├── .instructions.md ← 個別・レガシー用（廃止推奨） ├── src/ └── package.json

Skill 読み込みパスのカスタマイズ

chat.agentSkillsLocations 設定

GitHub Copilot が Skill を検索する優先順位を カスタマイズしたい場合は、.vscode/settings.json の chat.agentSkillsLocations で パス一覧を直接指定できます。推奨は ${workspaceFolder}/.github/skills を最優先に設定することです。

設定方法

1. プロジェクトの .vscode/settings.json を開く

my-project/ ├── .vscode/ │ └── settings.json ← ここに追記 ├── .github/ │ └── skills/ ← Skill をこの位置に配置 ├── src/ └── package.json

2. chat.agentSkillsLocations を設定

// .vscode/settings.json { "chat.agentSkillsLocations": [ "${workspaceFolder}/.github/skills" ] }

パス指定の種類

指定方法 意味 例
${workspaceFolder} プロジェクトルート ${workspaceFolder}/.github/skills
${userHome} ホームディレクトリ（フォールバック用） ${userHome}/.claude/skills
絶対パス フルパス指定 /Users/myname/projects/skills

優先順位

複数のパスを指定した場合、上から順に検索されます：

【GitHub Copilot 推奨】 1. ${workspaceFolder}/.github/skills ← チーム共有（最優先） ↓ （見つからなければ） 2. ${userHome}/.claude/skills ← 個人用（フォールバック）

推奨構成

.github/skills に Skill を配置

【推奨ディレクトリ構成】 my-project/ ├── .github/ │ ├── skills/ │ │ ├── react-best-practices/ │ │ │ ├── SKILL.md │ │ │ ├── metadata.json │ │ │ └── AGENTS.md │ │ ├── deploy-to-vercel/ │ │ │ ├── SKILL.md │ │ │ └── scripts/ │ │ └── composition-patterns/ │ │ ├── SKILL.md │ │ └── rules/ │ ├── workflows/ │ ├── ISSUE_TEMPLATE/ │ └── copilot-instructions.md ├── .vscode/ │ └── settings.json ├── src/ └── package.json 【settings.json の設定】 { "chat.agentSkillsLocations": [ "${workspaceFolder}/.github/skills" ] } 理由: • Git で管理でき、チーム全員が同じ Skill を使用可能 • GitHub 標準の設定位置（workflows, ISSUE_TEMPLATE と同じ） • Vercel や大規模プロジェクトの推奨パターン

フォールバック設定： 個人用グローバル Skill（オプション）

【個人開発用のグローバル設定】 複数プロジェクト間で共有する個人用 Skill が必要な場合： 1. ホームディレクトリに配置 ~/.claude/skills/ 2. .vscode/settings.json に追加 { "chat.agentSkillsLocations": [ "${workspaceFolder}/.github/skills", "${userHome}/.claude/skills" ] } 推奨: .github/skills をメイン構成に、~/.claude/skills は フォールバック・個人用のみとしてください。

AGENTS.md の活用

AGENTS.md の役割

各 Skill に含まれる AGENTS.md は、AI エージェント向けの詳細ドキュメントです。SKILL.md だけでなく、AGENTS.md も読み込まれることがあります。

参照方法

場面 参照先 詳細
初回のSkill化作 SKILL.md のみ 軽量な起動・判定
Skill を実際に使用 SKILL.md + AGENTS.md 詳細ルール・例まで参照
複雑な問題を解決 AGENTS.md の詳細セクション 高度なパターン・ケース

AGENTS.md の確認方法

  1. ローカルで確認
    # Skill の AGENTS.md を確認 cat .github/skills/react-best-practices/AGENTS.md # VS Code で直接開く code .github/skills/react-best-practices/AGENTS.md
  2. Copilot チャット内で参照
    Copilot チャット → 「@symbols react-best-practices」で Skill 内の記号を検索
  3. Skill ディレクトリをワークスペースに追加
    # VS Code で開く code .github/skills/react-best-practices/ # AGENTS.md、rules/ などの内容を直接参照可能

複数 Skill の統合管理

複数 Skill を効果的に運用するコツ

複数の Skill を同時に使用する場合の優先度管理と、コンフリクト回避について説明します。

Skill 優先度の設定

# .github/copilot-instructions.md での優先度設定例 ## Skill Priority (from highest to lowest) 1. team-naming-conventions # 社内ルール最優先 2. react-best-practices # Vercel ガイドライン 3. web-design-guidelines # UI/UX ガイドライン 4. composition-patterns # コンポーネント設計 ## Conflict Resolution - When multiple skills recommend different approaches, apply team-naming-conventions first

複数 Skill 運用のベストプラクティス

項目 推奨方法
有効/無効の切り替え copilot-instructions.md または AGENTS.md で「Always use」と「Optional」に分ける
バージョン管理 各 Skill の metadata.json で version を確認、要件に応じて更新
テスト・検証 新規 Skill 追加後、簡単なタスクで動作確認
ドキュメント管理 README.md にチーム内で利用している Skill リストを記載

トラブルシューティング

Skill が認識されない場合

  1. SKILL.md の frontmatter を確認（name, description が必須）
  2. ディレクトリ構造が正しいか確認（.github/skills/ に配置）
  3. VS Code を再起動してキャッシュをクリア
  4. .vscode/settings.json の chat.agentSkillsLocations が正しく設定されているか確認
