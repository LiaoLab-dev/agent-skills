📌 ユースケース 1: React コンポーネント最適化

🎯 シナリオ
既存の React コンポーネントが遅い。GitHub Copilot の助言を得ながら最適化したい。

対応する Skill

Skill 説明 優先度
react-best-practices React/Next.js パフォーマンス最適化 高
web-design-guidelines Web UI/UX ガイドライン 中
You: "このコンポーネント遅いんです、最適化してもらえますか？ ↓ Copilot: 1️⃣ react-best-practices Skill を自動起動 2️⃣ コード分析 → waterfalls、bundle size、re-renders をチェック 3️⃣ 見つかった改善点: ❌ Promise.all がない（waterfall） ❌ styled-components が毎回インポート ❌ useCallback がない（不要な再レンダ） 4️⃣ 改善コード + 説明をセット提示

実装ステップ

  1. VS Code で React ファイルを開く
  2. Copilot チャットで「このコンポーネント最適化して」と指示
  3. Copilot が意図的に react-best-practices ルールを適用
  4. パフォーマンス改善例がコード形式で提示される

📌 ユースケース 2: デプロイ自動化

🎯 シナリオ
開発が終わった。アプリを Vercel にデプロイしたい。手作業でなく AI に任せたい。

対応する Skill

Skill 説明 優先度
deploy-to-vercel Vercel へのデプロイ自動化 高
You: "Deploy my app" ↓ Copilot: 1️⃣ deploy-to-vercel Skill を検出 2️⃣ デプロイ方法を自動判定: - git remote + .vercel 設定 → git push - CLI 認証済み → vercel deploy - その他 → デプロイスクリプト実行 3️⃣ デプロイ実行 4️⃣ 完了： Preview URL: https://my-app-xyz.vercel.app Claim URL: https://vercel.com/claim-deployment?code=...

実装ステップ

  1. Copilot チャットで「Deploy my app」と入力
  2. Copilot が deploy-to-vercel インテント を検出
  3. 自動でデプロイフロー実行
  4. 成功時にプレビュー URL が返される

📌 ユースケース 3: 複数 Skill の組み合わせ

🎯 シナリオ
UI コンポーネント作成 → パフォーマンス最適化 → デプロイまで一連で行いたい。

発動する Skill 群

順序 Skill 役割 出力例
1️⃣ composition-patterns コンポーネント設計 Compound Component 推奨
2️⃣ react-best-practices パフォーマンス memo() + useCallback
3️⃣ web-design-guidelines UI/UX/アクセシビリティ aria-label + focus states
4️⃣ deploy-to-vercel デプロイ URL 生成
User: "Create a Button component, optimize it, and deploy" ↓ Copilot: 1️⃣ composition-patterns をロード → Compound Component 設計 2️⃣ react-best-practices をロード → メモ化、最適化 3️⃣ web-design-guidelines をロード → アクセシビリティチェック 4️⃣ すべてを統合したコンポーネント生成 5️⃣ deploy-to-vercel をロード → 自動デプロイ 6️⃣ プレビュー URL と Claim URL を返却

実装ステップ

  1. .instructions.md で複数 Skill を有効化
  2. Copilot に複合タスク指示を出す
  3. 複数 Skill が自動で連携
  4. 統合されたコード + デプロイURL が返される

📌 ユースケース 4: チーム固有ルールの自動適用

🎯 シナリオ
チーム内で「すべてのコンポーネント名は VerbNoun パターン」という規則がある。Copilot にこれを自動で守らせたい。

実装方法

手段 方法 効果
.instructions.md Team Naming Conventions Skill として記載 永遠に適用
README.md コーディング規則をドキュメント化 チーム内共有
カスタム Skill チーム固有の Skill を作成 再利用可能
# .instructions.md の例 ## Team Naming Conventions (MUST) - Component names: VerbNoun (ShowModal, FetchUser, RenderChart) - Functions: camelCase with action verbs (parseJSON, validateEmail) - NG: Boolean-only names (Modal, data_loader) Result: ↓ Copilot は自動的に VerbNoun パターンのコンポーネント名を生成

実装ステップ

  1. チーム Skill（team-naming-conventions など）を作成
  2. .instructions.md に登録
  3. Copilot 使用時に自動適用
  4. コード レビュー時に命名規則の手直しが不要に

💡 Skill 使用のコツ

✅ 効果的な指示文
  • ✅ 「このコンポーネント、パフォーマンス最適化してほしい」
  • ✅ 「Deploy my app」
  • ✅ 「Create a Button component with best practices」
  • ❌ 「コード書いて」（非具体的）
  • ❌ 「最適化して」（どの Skill を使うか曖昧）
⚙️ 複数 Skill の優先度管理
.instructions.md で Priority を指定すると、コンフリクト時に優先度の高い Skill が優先されます。
## Skill Priority 1. team-rules # 社内ルール最優先 2. react-best-practices 3. web-design-guidelines
🔄 Skill の有効/無効切り替え
プロジェクトによって必要な Skill が異なる場合、.instructions.md でプロジェクト固有に切り替える。
