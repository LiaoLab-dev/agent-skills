セットアップ前の確認

環境確認チェックリスト

以下の環境が整っていることを確認してから進めてください。

必須環境

項目 確認方法 最小要件
VS Code code --version v1.80 以上
GitHub Copilot 拡張機能 VS Code 拡張機能タブで確認 最新版
GitHub アカウント VS Code でサインイン 有効なアカウント
インターネット接続 ブラウザで確認 常に接続

確認手順

  1. VS Code のバージョン確認
    # ターミナルで実行 code --version # 出力例: 1.90.0 # v 1.80 以上であれば OK
  2. GitHub Copilot の拡張機能確認
    VS Code を開く → 左サイドバーの「拡張機能」アイコン → 「copilot」で検索 → 「GitHub Copilot」があれば OK
  3. GitHub へのサインイン
    VS Code 左下のアカウントアイコン → 「Sign in with GitHub」をクリック

Skill ディレクトリの設定

Skill をどこに配置するか

VS Code が Skill ファイルを認識するための推奨ディレクトリ構成を説明します。

推奨ディレクトリ構成

home/ ├── .claude/ │ ├── skills/ │ │ ├── react-best-practices/ │ │ │ ├── SKILL.md │ │ │ ├── metadata.json │ │ │ └── AGENTS.md │ │ ├── deploy-to-vercel/ │ │ │ ├── SKILL.md │ │ │ └── scripts/ │ │ └── composition-patterns/ │ │ ├── SKILL.md │ │ └── rules/ │ └── .instructions.md │ └── projects/ └── my-project/

ディレクトリ配置のステップ

  • Step 1
    .claude ディレクトリを作成
    # macOS / Linux mkdir -p ~/.claude/skills # Windows PowerShell New-Item -Path $env:USERPROFILE\.claude\skills -ItemType Directory
  • Step 2
    agent-skills をコピー
    # agent-skills リポジトリから skills をコピー cp -r agent-skills/skills/* ~/.claude/skills/ # 確認 ls ~/.claude/skills/ # 出力例: # react-best-practices/ deploy-to-vercel/ composition-patterns/
  • Step 3
    VS Code で認識確認
    VS Code を開く → コマンドパレット（Ctrl+Shift+P）→ 「Copilot: Open Slash Commands」を実行 → 登録された Skill が表示されるかチェック

.instructions.md の設定

.instructions.md とは

VS Code でエージェント（GitHub Copilot・Claude Code）の動作を永続的にカスタマイズするファイルです。ユーザーの好みやコーディング規則をエージェントに永遠に教え続けます。

推奨される .instructions.md の内容

# Agent Skill Preferences ## Always use these skills: 1. react-best-practices 2. composition-patterns 3. web-design-guidelines ## Coding Standards - Follow TypeScript strict mode - Use functional components with hooks - Prefer composition over inheritance - Add JSDoc comments for exports ## Code Style - Use 2 spaces for indentation - Follow Prettier formatting - Add type annotations for all functions - Use semantic HTML elements ## Performance - Always optimize for bundle size - Implement code splitting - Consider Lighthouse scores - Profile before optimizing

設定ファイルの作成手順

  1. テキストエディタで作成
    # ~/.claude/.instructions.md を新規作成 # 上のテンプレートをコピー＆ペースト
  2. プロジェクト全体に適用
    VS Code でプロジェクトを開く → .vscode/settings.json で指定
  3. VS Code の認識確認
    Copilot チャット → 「@instructions」でファイルが参照されているかチェック

プロジェクト固有のファイル配置

my-project/ ├── .vscode/ │ └── settings.json ├── .instructions.md ← プロジェクト固有の指示 ├── src/ │ └── components/ ├── package.json └── README.md

AGENTS.md の活用

AGENTS.md の役割

各 Skill に含まれる AGENTS.md は、AI エージェント向けの詳細ドキュメントです。SKILL.md だけでなく、AGENTS.md も読み込まれることがあります。

参照方法

場面 参照先 詳細
初回のSkill化作 SKILL.md のみ 軽量な起動・判定
Skill を実際に使用 SKILL.md + AGENTS.md 詳細ルール・例まで参照
複雑な問題を解決 AGENTS.md の詳細セクション 高度なパターン・ケース

AGENTS.md の確認方法

  1. ローカルで確認
    # Skill の AGENTS.md を確認 cat ~/.claude/skills/react-best-practices/AGENTS.md # VS Code で直接開く code ~/.claude/skills/react-best-practices/AGENTS.md
  2. Copilot チャット内で参照
    Copilot チャット → 「@symbols react-best-practices」で Skill 内の記号を検索
  3. Skill ディレクトリをワークスペースに追加
    # VS Code で開く code ~/.claude/skills/react-best-practices/ # AGENTS.md、rules/ などの内容を直接参照可能

複数 Skill の統合管理

複数 Skill を効果的に運用するコツ

複数の Skill を同時に使用する場合の優先度管理と、コンフリクト回避について説明します。

Skill 優先度の設定

# ~/.claude/.instructions.md での優先度設定例 ## Skill Priority (from highest to lowest) 1. team-naming-conventions # 社内ルール最優先 2. react-best-practices # Vercel ガイドライン 3. web-design-guidelines # UI/UX ガイドライン 4. composition-patterns # コンポーネント設計 ## Conflict Resolution - When multiple skills recommend different approaches, apply team-naming-conventions first

複数 Skill 運用のベストプラクティス

項目 推奨方法
有効/無効の切り替え .instructions.md で「Always use」と「Optional」に分ける
バージョン管理 各 Skill の metadata.json で version を確認、要件に応じて更新
テスト・検証 新規 Skill 追加後、簡単なタスクで動作確認
ドキュメント管理 README.md にチーム内で利用している Skill リストを記載

トラブルシューティング

Skill が認識されない場合

  1. SKILL.md の frontmatter を確認（name, description が必須）
  2. ディレクトリ構造が正しいか確認
  3. VS Code を再起動してキャッシュをクリア
  4. ~/.claude/skills/ の権限を確認
