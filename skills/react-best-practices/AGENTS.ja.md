# AGENTS.md

このファイルは、このリポジトリ内のコードで作業する場合、AIコーディングエージェント（Claude Code、Cursor、Copilotなど）にガイダンスを提供します。

## リポジトリの概要

Vercelのデプロイメントで作業するためのClaude.aiおよびClaude Code向けのスキルのコレクション。スキルはエージェントの機能を拡張するためにパッケージ化された命令とスクリプトです。

## 新しいスキルを作成する

### ディレクトリ構造

```
skills/
  {skill-name}/           # ケバブケースのディレクトリ名
    SKILL.md              # 必須：スキル定義
    scripts/              # 必須：実行可能なスクリプト
      {script-name}.sh    # Bashスクリプト（推奨）
  {skill-name}.zip        # 必須：配布用にパッケージ化
```

### 命名規則

- **スキルディレクトリ**: `kebab-case`（例：`vercel-deploy`、`log-monitor`）
- **SKILL.md**: 常に大文字、常にこの正確なファイル名
- **スクリプト**: `kebab-case.sh`（例：`deploy.sh`、`fetch-logs.sh`）
- **Zipファイル**: ディレクトリ名と完全に一致する必要があります：`{skill-name}.zip`

### SKILL.mdフォーマット

```markdown
---
name: {skill-name}
description: {スキルをいつ使用するかを説明する1文。「アプリをデプロイして」、「ログをチェック」などのトリガーフレーズを含めます。}
---

# {スキルタイトル}

{スキルが何をするかについての簡潔な説明。}

## 仕組み

{スキルのワークフローを説明する番号リスト}

## 使用方法

\`\`\`bash
bash /mnt/skills/user/{skill-name}/scripts/{script}.sh [args]
\`\`\`

**引数：**
- `arg1` - 説明（デフォルトはX）

**例：**
{一般的な使用パターンを2〜3個表示}

## 出力

{ユーザーが表示される出力例}

## ユーザーに結果を提示する

{結果をユーザーに提示する際にClaudeがどのようにフォーマットすべきかのテンプレート}

## トラブルシューティング

{一般的な問題と解決策、特にネットワーク/権限エラー}
```

### コンテキスト効率のためのベストプラクティス

スキルはオンデマンドで読み込まれます— スキル名と説明のみが起動時に読み込まれます。完全な`SKILL.md`は、エージェントがスキルが関連していると判断した場合のみコンテキストに読み込まれます。コンテキストの使用を最小化するために：

- **SKILL.mdを500行以下に保つ** — 詳細なリファレンス資料は別のファイルに圧縮
- **具体的な説明を作成** — エージェントがスキルをいつ有効化するかを正確に知るのに役立ちます
- **段階的な公開を使用** — 必要な場合にのみ読まれるサポートファイルを参照
- **インラインコードよりスクリプトを優先** — スクリプト実行はコンテキストを消費しません（出力のみ）
- **ファイル参照は1レベル深さまで機能** — SKILL.mdからサポートファイルに直接リンク

### スクリプト要件

- `#!/bin/bash`シェバングを使用
- fail-fastの動作に`set -e`を使用
- ステータスメッセージをstderrに書き込み：`echo "Message" >&2`
- マシン可読出力（JSON）をstdoutに書き込み
- 一時ファイル用のクリーンアップトラップを含める
- スクリプトパスを`/mnt/skills/user/{skill-name}/scripts/{script}.sh`として参照

### Zipパッケージを作成する

スキルを作成または更新した後：

```bash
cd skills
zip -r {skill-name}.zip {skill-name}/
```

### エンドユーザーインストール

ユーザーに2つのインストール方法をドキュメント化します：

**Claude Code:**
```bash
cp -r skills/{skill-name} ~/.claude/skills/
```

**claude.ai:**
スキルをプロジェクトナレッジに追加するか、SKILL.mdコンテンツを会話に貼り付けます。

スキルがネットワークアクセスを必要とする場合、`claude.ai/settings/capabilities`で必要なドメインを追加するようユーザーに指示します。
